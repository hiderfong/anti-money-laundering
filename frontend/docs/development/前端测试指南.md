# 前端测试指南

## 1. 单元测试

前端使用 Vitest 作为单元测试框架。

### 运行单元测试

```bash
cd frontend

# 运行所有测试
npm run test

# 监听模式
npm run test:watch

# 查看覆盖率
npm run test:coverage
```

### 编写单元测试

测试文件放在 `src/` 下与被测文件同目录，命名为 `*.test.ts`。

```
src/
  utils/
    format.ts
    format.test.ts
  stores/
    user.ts
    user.test.ts
```

### 覆盖率要求

| 指标       | 最低要求 |
|-----------|---------|
| 语句覆盖   | 60%     |
| 分支覆盖   | 50%     |
| 函数覆盖   | 60%     |
| 行覆盖     | 60%     |

## 2. E2E 测试

E2E 测试使用 bash 脚本，无需额外框架依赖。

### 前端可用性测试

```bash
# 确保前端服务已启动
cd frontend && npm run dev

# 另一个终端运行
bash frontend/scripts/frontend-e2e.sh
```

测试内容包括：
- 前端服务连通性
- 登录页面可访问性
- 静态资源（JS/CSS）加载
- API 代理转发
- SPA 路由功能

### 后端 API 测试

```bash
bash scripts/e2e-test.sh
```

## 3. 手动测试清单

### 登录流程
- [ ] 访问任意页面自动跳转登录页
- [ ] 输入错误凭据显示错误提示
- [ ] 登录成功跳转仪表盘
- [ ] 登出后跳转登录页

### 权限控制
- [ ] 无权限用户访问受保护页面跳转 403
- [ ] 访问不存在的路径显示 404 页面
- [ ] 管理员可访问所有页面

### 响应式
- [ ] 1920px 桌面正常显示
- [ ] 1366px 笔记本正常显示
- [ ] 768px 平板正常显示
