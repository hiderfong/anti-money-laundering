## 二、《反洗钱法》（2024年修订版）核心要点

> 2024年11月8日通过，2025年1月1日起施行

### 2.1 适用范围扩展

- **特定非金融机构**首次被纳入反洗钱义务主体范围，包括：
  - 房地产开发企业、房地产中介机构
  - 贵金属交易商、宝石交易商
  - 会计师事务所、律师事务所
  - 公证机构
- **产品设计影响**：系统需考虑对特定非金融机构交易数据的对接能力

### 2.2 金融机构核心义务（产品设计必读）

#### 义务一：客户身份识别（KYC/CDD）

| 要求 | 具体内容 | 系统功能需求 |
|------|---------|------------|
| 客户身份登记 | 姓名/名称、住所、身份证件种类及号码、有效期等 | 客户信息管理模块 |
| 身份核实 | 通过可靠渠道核实客户身份 | 身份核验接口 |
| 受益所有人识别 | 识别并核实最终受益所有人 | 穿透识别模块 |
| 持续识别 | 持续关注客户身份信息变化 | 定期复核提醒 |
| 重新识别 | 客户行为异常时重新识别 | 触发条件配置 |

#### 义务二：大额交易和可疑交易报告

**大额交易报告标准（保险业适用）：**

| 交易类型 | 报告门槛 | 报告时限 |
|---------|---------|---------|
| 现金交易 | 单笔或当日累计人民币5万元以上（含）或等值外币 | 交易发生后5个工作日内 |
| 转账交易（自然人） | 单笔或当日累计人民币200万元以上（含）或等值外币 | 交易发生后5个工作日内 |
| 转账交易（法人/其他组织） | 单笔或当日累计人民币200万元以上（含）或等值外币 | 交易发生后5个工作日内 |
| 跨境交易 | 自然人单笔或当日累计20万元以上（含）或等值外币 | 交易发生后5个工作日内 |
| 跨境交易 | 法人单笔或当日累计200万元以上（含）或等值外币 | 交易发生后5个工作日内 |

**可疑交易报告要求：**
- 发现或有合理理由怀疑客户交易涉嫌洗钱的，应在**10个工作日内**提交可疑交易报告
- 涉及恐怖活动的，应**立即**报告
- 可疑交易报告不得向客户或无关人员透露

#### 义务三：客户身份资料和交易记录保存

| 保存内容 | 保存期限 | 保存要求 |
|---------|---------|---------|
| 客户身份资料 | 自业务关系结束之日起至少**5年** | 完整、准确 |
| 交易记录 | 自交易记账之日起至少**5年** | 可还原交易全貌 |
| 可疑交易报告及分析资料 | 自报告之日起至少**5年** | 保密 |

#### 义务四：内部控制制度

- 设立反洗钱专门机构或指定内设机构
- 建立客户风险等级划分制度
- 建立反洗钱名单监控机制
- 开展反洗钱培训和宣传
- 配合反洗钱调查和检查

## 三、保险业反洗钱特别规定

### 3.1 《保险业反洗钱工作管理办法》要点

**保险公司应履行的特殊义务：**

1. **产品洗钱风险评估**
   - 对保险产品进行洗钱风险评估
   - 高风险产品应加强管控措施
   - 产品设计阶段即应考虑洗钱风险

2. **高风险产品类型（重点关注）**
   - 具有现金价值的人寿保险
   - 投资连结保险
   - 万能保险
   - 年金保险
   - 团体保险（大额保单）
   - 保费趸缴型产品
   - 保单贷款业务

3. **销售渠道管理**
   - 代理渠道的反洗钱义务传导
   - 银保渠道的特别管控要求
   - 互联网渠道的身份识别强化

### 3.2 保险业客户风险等级划分

| 风险等级 | 划分标准 | 管控措施 |
|---------|---------|---------|
| 低风险 | 客户身份明确、交易正常、无异常特征 | 常规监测 |
| 中风险 | 存在一定风险因素但尚不构成高风险 | 加强关注 |
| 高风险 | 来自高风险国家/地区、PEP（政治公众人物）、异常交易行为 | 强化尽职调查、提高监测频率、管理层审批 |
| 禁止类 | 涉及联合国制裁名单 | 拒绝建立业务关系 |

### 3.3 保险业可疑交易识别参考指标

**投保环节：**
- 客户拒绝提供身份信息或提供虚假信息
- 代为投保且无法说明合理理由
- 投保金额与客户经济状况明显不符
- 短期内在多家保险公司投保大额保单
- 要求以现金方式支付大额保费
- 投保后短期内要求退保或保单贷款
- 境外人士投保境内保险且无合理商业理由

**理赔环节：**
- 理赔金额与实际损失明显不符
- 理赔资金要求转入非投保人/被保险人账户
- 伪造或夸大保险事故
- 短期内多次小额理赔

**退保环节：**
- 犹豫期后立即退保
- 退保资金要求转入第三方账户
- 大额保单退保后短期内重新投保

## 四、客户身份识别（KYC/CDD）详细要求

### 4.1 个人客户身份识别要素

**基本身份信息：**
- 姓名、性别、国籍、职业
- 住所地或经常居住地
- 联系方式（电话、邮箱）
- 身份证件种类、号码和有效期限
- 税收居民身份信息（CRS要求）

**加强型尽职调查（EDD）触发条件：**
- 客户来自FATF高风险或不合作国家/地区
- 客户为政治公众人物（PEP）及其关系密切人
- 客户被列入反洗钱监控名单
- 客户行为异常或交易可疑
- 非面对面建立业务关系

### 4.2 法人/机构客户身份识别要素

**基本信息：**
- 名称、注册地址、经营范围
- 成立日期、注册资本
- 法定代表人/负责人信息
- 股权结构信息
- 资金来源和用途

**受益所有人识别：**
- 直接或间接持有法人25%以上股权或表决权的自然人
- 直接或间接享有法人25%以上权益的自然人
- 对法人实施最终有效控制的自然人
- 多层穿透至最终自然人

### 4.3 保险业特殊身份识别场景

| 场景 | 身份识别要求 |
|------|------------|
| 投保人 | 完整KYC，与保费规模匹配 |
| 被保险人 | 基本身份信息，大额保单需完整KYC |
| 受益人 | 身份信息登记，理赔时核实 |
| 保单持有人变更 | 对新持有人进行KYC |
| 保单贷款 | 识别贷款人身份 |
| 退保 | 核实退款账户与投保人一致 |

## 五、名单监控（制裁筛查）要求

### 5.1 必须监控的名单类型

| 名单类型 | 来源 | 更新频率要求 |
|---------|------|------------|
| 联合国安理会制裁名单 | UN Security Council | 实时/次日 |
| OFAC SDN名单 | 美国财政部 | 实时/次日 |
| EU制裁名单 | 欧盟 | 实时/次日 |
| 中国公安部恐怖活动组织/人员名单 | 公安部 | 即时更新 |
| 人民银行反洗钱监控名单 | 人民银行 | 即时更新 |
| FATF高风险国家/地区 | FATF | 定期更新 |

### 5.2 名单监控节点

- **客户准入**：建立业务关系时必须筛查
- **定期复查**：至少每季度对存量客户进行一次名单筛查
- **名单更新**：新名单发布后对存量客户进行增量筛查
- **高风险客户**：提高筛查频率（月度或更频繁）

### 5.3 命中处理流程

```
名单命中 → 人工复核 → 确认/排除
                         ├── 排除：记录原因，继续业务
                         └── 确认：冻结资产/拒绝业务 → 报告监管
```

## 六、可疑交易报告（STR）制度

### 6.1 报告流程

```
系统自动识别/人工发现可疑交易
    ↓
反洗钱岗初步分析
    ↓
填写可疑交易报告要素
    ↓
内部审批（反洗钱负责人）
    ↓
通过反洗钱监测分析系统报送人民银行
    ↓
配合后续调查（如有）
```

### 6.2 可疑交易报告必填要素

- 涉及的客户基本信息
- 账户信息（保单号等）
- 交易信息（时间、金额、方式）
- 可疑交易特征描述
- 初步分析意见
- 已采取的措施

### 6.3 保密要求

- 不得向客户透露已报送可疑交易报告
- 不得向无关人员透露报告内容
- 报告资料按机密级别管理

## 八、处罚条款摘要

### 8.1 《反洗钱法》规定的行政处罚

| 违法行为 | 处罚对象 | 处罚标准 |
|---------|---------|---------|
| 未按规定履行客户身份识别义务 | 机构 | 20万-200万元罚款；情节严重，责令停业整顿或吊销许可证 |
| 未按规定报送大额/可疑交易报告 | 机构 | 20万-200万元罚款 |
| 未按规定保存客户资料和交易记录 | 机构 | 20万-200万元罚款 |
| 未按规定建立反洗钱内部控制制度 | 机构 | 20万-200万元罚款 |
| 上述行为的直接责任人员 | 个人 | 1万-50万元罚款；情节严重，取消任职资格 |
| 泄露反洗钱信息 | 机构/个人 | 依法追究责任 |

### 8.2 刑事责任

| 罪名 | 量刑标准 |
|------|---------|
| 洗钱罪（刑法191条） | 没收犯罪所得及其收益，处五年以下有期徒刑或拘役，并处或单处洗钱金额5%-20%罚金；情节严重的，处五年以上十年以下有期徒刑，并处洗钱金额5%-20%罚金 |
| 掩饰、隐瞒犯罪所得罪（刑法312条） | 三年以下有期徒刑、拘役或管制，并处或单处罚金；情节严重的，三年以上七年以下有期徒刑，并处罚金 |

## 九、产品设计映射：法规要求 → 系统功能

### 9.1 核心功能模块对照表

| 法规要求 | 对应系统模块 | 优先级 |
|---------|------------|-------|

| 可疑交易报告 | 可疑交易监测与报告模块 | P0 |

| 受益所有人识别 | 穿透识别模块 | P1 |
| PEP识别 | 政治公众人物筛查模块 | P1 |

| 反洗钱培训管理 | 培训管理模块 | P2 |

### 9.2 关键业务规则清单

1. **客户准入拦截**：命中制裁名单 → 自动拦截，禁止建立业务关系
2. **大额交易自动报告**：交易金额达到阈值 → 自动生成大额交易报告
3. **可疑交易预警**：触发可疑指标 → 生成预警工单，人工分析确认后报送
4. **客户风险动态调整**：根据交易行为变化 → 自动调整客户风险等级
5. **定期复核提醒**：高风险客户至少每年复核一次，中风险至少每两年
6. **资料保存期限**：自动归档，确保5年内可查询
7. **交易链路追溯**：保单全生命周期交易可追溯

## 十、数据报送接口规范

### 10.1 人民银行反洗钱监测分析系统

- **报送方式**：通过反洗钱监测分析系统（AML-MIS）在线报送
- **数据格式**：XML格式，符合接口规范
- **报送频率**：大额交易T+5工作日，可疑交易发现后10个工作日内
- **系统对接**：需申请数字证书，通过专线或VPN接入

### 10.2 金融监管总局报送

- 反洗钱工作报告（年度）
- 洗钱风险自评估报告（年度）
- 重大洗钱风险事件即时报告
