# 反洗钱系统产品需求文档（PRD）

> 项目名称：保险公司反洗钱管理系统（AML System）  
> 版本：v1.0  
> 日期：2025年5月  
> 状态：需求细化阶段

---

## 一、产品概述

### 1.1 产品定位

面向保险公司内部使用的反洗钱合规管理系统，覆盖客户身份识别、交易监测、名单筛查、可疑交易报告、案件管理、监管报送等全流程，满足《反洗钱法》（2024修订版）及金融监管总局对保险业的反洗钱监管要求。

### 1.2 核心目标

| 目标 | 衡量指标 |
|------|---------|
| 合规达标 | 满足人民银行和金融监管总局全部强制性要求 |
| 降低风险 | 可疑交易识别覆盖率 >= 95% |
| 提升效率 | 人工审核效率提升 50% 以上（相比手工模式） |
| 降低误报 | 预警误报率 <= 70%（行业平均 85%-95%） |
| 审计可追溯 | 全链路操作日志，支持监管检查回溯 |

### 1.3 用户角色定义

| 角色 | 职责 | 使用频率 |
|------|------|---------|
| 反洗钱专员（AML Officer） | 日常预警审核、可疑交易分析、报告撰写 | 每日 |
| 反洗钱主管（AML Manager） | 案件审批、报告签发、策略调整、团队管理 | 每日 |
| 合规负责人（Compliance Director） | 监管对接、制度审批、风险自评估 | 每周 |
| 系统管理员（System Admin） | 规则配置、名单维护、用户权限、系统运维 | 每周 |
| 业务操作员（Business Operator） | 客户信息录入、保单操作时触发 KYC 流程 | 按业务量 |
| 审计人员（Auditor） | 查询操作日志、导出审计报告 | 按需 |
| 高级管理层（Executive） | 查看风险仪表盘、审批重大事项 | 每月 |

## 二、功能模块详细需求

### 模块一：客户身份识别（KYC/CDD）

#### 2.1.1 业务场景

| 场景 | 触发时机 | 识别深度 |
|------|---------|---------|
| 新客户投保 | 建立业务关系时 | 完整 KYC |
| 存量客户信息变更 | 保全操作时 | 重新识别 |
| 客户风险等级提升 | 系统触发 | 加强型尽职调查（EDD） |
| 保单受益人变更 | 保全操作时 | 受益人 KYC |
| 大额理赔 | 理赔金额超阈值时 | 受益人身份核实 |
| 定期复核 | 高风险客户每年 / 中风险每两年 | 信息更新确认 |

#### 2.1.2 功能清单

**F-KYC-001 个人客户信息管理**
- 录入字段：姓名、性别、国籍、民族、出生日期、住所地、经常居住地
- 证件类型：身份证、护照、港澳通行证、台湾通行证、军官证、其他
- 证件信息：证件号码、签发机关、有效期起止
- 联系信息：手机号码、固定电话、电子邮箱
- 职业信息：职业类别、工作单位、职务
- 税收居民身份：中国税收居民 / 非居民 / 既是中国也是其他税收居民
- 证件影像：支持证件正反面扫描件上传、OCR 自动识别
- 必填校验：根据业务类型和交易金额动态调整必填字段

**F-KYC-002 法人/机构客户信息管理**
- 基本信息：名称、统一社会信用代码、注册地址、经营范围
- 成立信息：成立日期、注册资本、企业类型
- 法定代表人/负责人信息
- 联系信息
- 股权结构信息（至少两层穿透）
- 证件类型：营业执照、组织机构代码证、其他

**F-KYC-003 受益所有人识别**
- 自动识别标准：
  - 直接或间接持有 25% 以上股权的自然人
  - 直接或间接享有 25% 以上权益的自然人
  - 对法人实施最终有效控制的自然人
- 穿透层级：至少穿透至最终自然人，建议支持 5 层以上
- 股权结构可视化：树状图展示股权穿透关系
- 人工补充：对无法自动识别的情况支持人工标注

**F-KYC-004 身份核验**
- 身份证二要素核验（姓名 + 身份证号）
- 银行卡四要素核验（姓名 + 身份证号 + 卡号 + 手机号）
- 企业信息核验（企业名称 + 统一社会信用代码）
- 人脸识别活体检测（可选，高风险场景）
- 核验结果记录和异常标记

**F-KYC-005 客户风险等级划分**
- 风险维度：

| 维度 | 评分因素 | 权重 |
|------|---------|------|
| 客户属性 | 个人/法人、PEP、非居民 | 25% |
| 地域风险 | 注册地/国籍所在国风险等级 | 20% |
| 产品风险 | 产品洗钱风险等级 | 20% |
| 渠道风险 | 面对面/非面对面、代理渠道 | 15% |
| 交易行为 | 交易金额、频率、方式 | 20% |

- 风险等级：低风险（1-30分）、中风险（31-60分）、高风险（61-100分）
- 自动评级 + 人工调整
- 等级变更审批流程（高风险需主管审批）
- 复核周期提醒：低风险 3 年、中风险 2 年、高风险 1 年

**F-KYC-006 PEP（政治公众人物）筛查**
- 内置 PEP 名单库（国内 + 国际）
- 自动筛查：客户准入时和定期复查时
- PEP 分类：国内 PEP、国际 PEP、国际组织 PEP
- PEP 关系人识别：家庭成员、密切关系人
- PEP 客户自动标记为高风险，需强化尽职调查

### 模块二：制裁名单筛查

#### 2.2.1 名单源管理

**F-SCR-001 名单源接入**

| 名单类型 | 来源 | 更新频率 | 优先级 |
|---------|------|---------|-------|
| 联合国安理会制裁名单 | UN Security Council | 实时 | P0 |
| OFAC SDN 名单 | 美国财政部 | 每日 | P0 |
| EU 制裁名单 | 欧盟 | 每日 | P0 |
| 中国公安部恐怖活动名单 | 公安部 | 即时 | P0 |
| 人民银行反洗钱监控名单 | 人民银行 | 即时 | P0 |
| FATF 高风险国家/地区 | FATF | 季度 | P1 |
| 其他国家制裁名单 | 各国财政部 | 每日 | P2 |

**F-SCR-002 名单数据管理**
- 名单导入：支持 XML、CSV、JSON 格式
- 名单解析：自动解析姓名、别名、证件号、出生日期、国籍、地址
- 增量更新：支持全量和增量两种更新模式
- 版本管理：名单变更历史可追溯
- 有效性管理：过期名单自动标记

#### 2.2.2 筛查引擎

**F-SCR-003 实时筛查**
- 筛查节点：

| 节点 | 触发时机 | 筛查范围 |
|------|---------|---------|
| 客户准入 | 新建客户/投保 | 客户全量信息 |
| 信息变更 | 客户资料修改 | 变更字段 |
| 交易发起 | 付款方/收款方 | 交易对手信息 |
| 名单更新 | 新名单导入 | 存量客户增量筛查 |

- 响应时间：单次筛查 <= 200ms
- 并发能力：支持 500 TPS

**F-SCR-004 匹配算法**
- 精确匹配：证件号码、身份证号（权重 100%）
- 模糊匹配算法：
  - Jaro-Winkler（英文姓名，阈值可配，默认 0.85）
  - 拼音转换 + 编辑距离（中文姓名）
  - Soundex / Metaphone（发音相似）
  - N-gram（部分匹配）
- 组合匹配：姓名 + 出生日期 / 姓名 + 国籍
- 相似度评分：0-100 分，支持多级阈值配置
  - >= 95 分：自动命中，直接进入处理流程
  - 85-94 分：疑似命中，进入人工复核
  - < 85 分：不命中，记录日志

**F-SCR-005 命中处理**
- 人工复核工作台：
  - 展示匹配详情（客户信息 vs 名单信息逐字段对比）
  - 支持查看原始名单数据
  - 支持查看客户历史交易
- 复核结论：
  - 确认命中 → 冻结业务 + 上报
  - 误报排除 → 记录排除理由 + 加入白名单（可选）
  - 无法确认 → 升级至主管
- 白名单管理：
  - 白名单有效期（默认 1 年，可续期）
  - 白名单审批流程
  - 白名单定期复查

### 模块三：交易监测

#### 2.3.1 大额交易监测

**F-TXN-001 大额交易规则**

| 规则编号 | 交易类型 | 报告门槛 | 报告时限 |
|---------|---------|---------|---------|
| LR-001 | 现金交易（个人） | 单笔/当日累计 >= 5万元 | T+5 工作日 |
| LR-002 | 转账交易（个人） | 单笔/当日累计 >= 200万元 | T+5 工作日 |
| LR-003 | 转账交易（法人） | 单笔/当日累计 >= 200万元 | T+5 工作日 |
| LR-004 | 跨境交易（个人） | 单笔/当日累计 >= 20万元 | T+5 工作日 |
| LR-005 | 跨境交易（法人） | 单笔/当日累计 >= 200万元 | T+5 工作日 |

- 自动监测：交易数据实时接入，自动判断是否达到报告门槛
- 累计计算：同一客户当日同类型交易累计（支持跨保单）
- 自动生成报告：达到门槛自动生成大额交易报告草稿
- 报告审核：支持人工审核确认后提交
- 补报机制：漏报自动检测和补报提醒

**F-TXN-002 大额交易报告管理**
- 报告列表：按时间、客户、金额筛选
- 报告详情：交易信息、客户信息、报告状态
- 批量审核：支持批量确认/退回
- 报告导出：支持 XML（人民银行格式）和 Excel
- 报送状态追踪：已报送 / 待报送 / 已退回 / 已补报

#### 2.3.2 可疑交易监测

**F-TXN-003 可疑交易规则库**

**投保环节规则：**

| 规则编号 | 规则描述 | 风险等级 |
|---------|---------|---------|
| SR-001 | 客户拒绝提供身份信息或提供虚假信息 | 高 |
| SR-002 | 代为投保且无法说明合理理由 | 中 |
| SR-003 | 投保金额与客户经济状况明显不符（年收入的 N 倍以上） | 高 |
| SR-004 | 短期内在多家保险公司投保大额保单 | 高 |
| SR-005 | 要求以现金方式支付大额保费 | 中 |
| SR-006 | 境外人士投保境内保险且无合理商业理由 | 中 |
| SR-007 | 使用他人账户支付保费 | 中 |
| SR-008 | 投保高风险产品（趸缴、万能、投连）且金额异常 | 中 |

**保全环节规则：**

| 规则编号 | 规则描述 | 风险等级 |
|---------|---------|---------|
| SR-009 | 投保后短期内要求退保（犹豫期后 30 天内） | 高 |
| SR-010 | 退保资金要求转入非投保人账户 | 高 |
| SR-011 | 频繁变更受益人 | 中 |
| SR-012 | 频繁进行保单贷款和还款 | 中 |
| SR-013 | 大额加保且资金来源不明 | 中 |
| SR-014 | 保单质押后短期内解除质押 | 低 |

**理赔环节规则：**

| 规则编号 | 规则描述 | 风险等级 |
|---------|---------|---------|
| SR-015 | 理赔金额与实际损失明显不符 | 高 |
| SR-016 | 理赔资金要求转入非被保险人/受益人账户 | 高 |
| SR-017 | 伪造或夸大保险事故 | 高 |
| SR-018 | 短期内多次小额理赔 | 中 |
| SR-019 | 新保单短期内即发生理赔 | 中 |

**通用规则：**

| 规则编号 | 规则描述 | 风险等级 |
|---------|---------|---------|
| SR-020 | 客户被列入反洗钱监控名单 | 极高 |
| SR-021 | 客户为已知 PEP 或其关系人 | 高 |
| SR-022 | 交易涉及 FATF 高风险国家/地区 | 高 |
| SR-023 | 客户交易模式突然发生重大变化 | 中 |
| SR-024 | 客户无法合理解释资金来源 | 高 |
| SR-025 | 多个客户使用相同联系方式/地址 | 中 |

**F-TXN-004 规则引擎**
- 规则配置界面：
  - 规则新增/编辑/停用/删除
  - 规则条件组合（AND/OR）
  - 阈值参数可调
  - 生效时间设置
  - 规则优先级排序
- 规则类型：
  - 阈值规则（金额/次数/频率）
  - 速度规则（时间窗口内交易频率）
  - 聚合规则（累计金额/笔数）
  - 关联规则（多字段组合匹配）
  - 模式规则（交易序列模式）
- 规则测试：支持历史数据回放验证规则效果

**F-TXN-005 风险评分**
- 多维度加权评分：
  - 规则命中得分
  - 客户风险等级得分
  - 产品风险得分
  - 交易特征得分
- 综合风险分：0-100 分
- 预警阈值：
  - >= 80 分：高风险预警（红色）
  - 60-79 分：中风险预警（橙色）
  - 40-59 分：低风险预警（黄色）
  - < 40 分：正常（不生成预警）

**F-TXN-006 交易行为基线**
- 自动学习客户历史交易模式
- 基线维度：
  - 月均交易金额
  - 月均交易笔数
  - 常用交易渠道
  - 常用交易时间
  - 常用交易对手
- 偏离度检测：当前交易与基线的偏离程度
- 基线更新周期：每月自动更新

### 模块四：预警与案件管理

#### 2.4.1 预警管理

**F-ALT-001 预警生成**
- 自动生成：规则触发 + 风险评分达到阈值
- 手动创建：人工发现可疑情况手动录入
- 预警去重：相同客户相同规则短期内不重复生成
- 预警聚合：同一客户多个规则命中聚合为一个预警工单

**F-ALT-002 预警处理工作台**
- 预警列表：
  - 筛选条件：风险等级、状态、时间、客户、规则
  - 排序：按风险分、生成时间
  - 分页、批量操作
- 预警详情：
  - 触发规则及命中详情
  - 客户基本信息和风险等级
  - 关联交易列表
  - 客户历史预警记录
  - 身份核验结果
  - 名单筛查结果
- 处理动作：
  - 确认可疑 → 创建可疑交易报告
  - 排除预警 → 填写排除理由
  - 转交他人 → 选择接手人
  - 升级处理 → 提交主管审核
  - 补充信息 → 标记需补充材料

**F-ALT-003 预警分配**
- 自动分配规则：
  - 按客户所属机构分配
  - 按风险等级分配（高风险分配给资深专员）
  - 负载均衡（各专员工作量均衡）
- 手动分配：主管可手动调整
- 超时升级：未在规定时间内处理自动升级

#### 2.4.2 案件管理

**F-CAS-001 案件生命周期**

```
预警生成 → 预警审核 → 可疑确认 → 报告撰写 → 内部审批 → 监管报送
    ↓           ↓           ↓           ↓           ↓
  排除        排除        补充调查     退回修改     退回修改
                                                    ↓
                                              配合调查 → 结案归档
```

**F-CAS-002 案件信息管理**
- 案件编号（自动生成）
- 关联预警
- 涉及客户列表
- 涉及交易列表
- 案件状态
- 调查记录（时间线形式）
- 证据材料（附件上传）
- 处理意见
- 审批记录

**F-CAS-003 可疑交易报告（STR）生成**
- 报告模板：符合人民银行要求的标准格式
- 必填要素：
  - 涉及客户基本信息
  - 账户/保单信息
  - 交易信息（时间、金额、方式、对手方）
  - 可疑交易特征描述
  - 初步分析意见
  - 已采取的措施
- 报告审批流程：
  - 撰写人（反洗钱专员）→ 审核人（反洗钱主管）→ 签发人（合规负责人）
- 报告状态追踪：草稿 / 待审核 / 已审核 / 已报送 / 已退回

**F-CAS-004 保密管理**
- 案件信息权限隔离
- 操作日志记录
- 打印/导出水印
- 禁止向客户透露的提醒机制

### 模块五：监管报送

**F-RPT-001 大额交易报告报送**
- 数据格式：XML（符合人民银行接口规范）
- 报送方式：通过反洗钱监测分析系统（AML-MIS）报送
- 报送前校验：
  - 字段完整性校验
  - 数据格式校验
  - 逻辑一致性校验
- 报送状态管理：待报送 / 已报送 / 报送失败 / 已确认
- 失败重试：自动重试 + 人工介入

**F-RPT-002 可疑交易报告报送**
- 报送时限管理：发现后 10 个工作日内
- 紧急报送：涉及恐怖活动的立即报送
- 报送提醒：临近截止日期自动提醒

**F-RPT-003 向金融监管总局报送**
- 反洗钱工作报告（年度）
- 洗钱风险自评估报告（年度）
- 重大洗钱风险事件即时报告
- 报告模板管理

**F-RPT-004 报送日志**
- 报送记录完整保存（至少 5 年）
- 报送状态追踪
- 报送数据版本管理

### 模块六：产品洗钱风险评估

**F-PRD-001 产品风险评估模型**

| 评估维度 | 评分因素 | 权重 |
|---------|---------|------|
| 产品类型 | 人寿/财产/健康/意外 | 20% |
| 缴费方式 | 趸缴/期缴/灵活缴费 | 20% |
| 产品结构 | 是否有现金价值、投资功能 | 20% |
| 退保灵活性 | 犹豫期、退保手续费比例 | 15% |
| 受益人安排 | 是否可变更、是否为第三方 | 15% |
| 销售渠道 | 直销/代理/银保/互联网 | 10% |

- 产品风险等级：低 / 中 / 高
- 新产品上线前必须完成洗钱风险评估
- 产品风险等级变更审批流程

**F-PRD-002 高风险产品清单**

| 产品类型 | 风险等级 | 管控措施 |
|---------|---------|---------|
| 万能保险 | 高 | 强化 KYC、交易监测加严 |
| 投资连结保险 | 高 | 强化 KYC、交易监测加严 |
| 大额趸缴终身寿险 | 高 | 大额保费来源核实 |
| 年金保险 | 中高 | 退保和受益人变更监控 |
| 团体保险（大额） | 中高 | 投保单位背景调查 |
| 保单贷款 | 中 | 贷款资金用途监控 |

## 四、数据模型概要

### 4.1 核心实体关系

```
客户 (Customer) 1:N 保单 (Policy)
客户 (Customer) 1:N 交易 (Transaction)
客户 (Customer) 1:N 预警 (Alert)
客户 (Customer) 1:N 身份核验记录 (Verification)
客户 (Customer) N:N 名单命中 (WatchlistHit)
保单 (Policy) 1:N 交易 (Transaction)
预警 (Alert) 1:1 案件 (Case)
案件 (Case) 1:1 可疑交易报告 (STR)
案件 (Case) N:N 附件 (Attachment)
```

### 4.2 核心实体字段（概要）

**客户表 (t_customer)**
- customer_id, customer_type(个人/法人), name, id_type, id_number
- nationality, gender, birth_date, address, phone, email
- occupation, employer, annual_income
- risk_level(低/中/高), risk_score, risk_update_date
- pep_flag, sanctions_flag
- create_time, update_time, status

**保单表 (t_policy)**
- policy_id, customer_id, product_code, product_name
- product_risk_level, sum_insured, premium, payment_mode
- effective_date, expiry_date, status
- beneficiary_info (JSON)

**交易表 (t_transaction)**
- transaction_id, policy_id, customer_id
- transaction_type(投保/退保/理赔/保全/贷款)
- amount, currency, payment_method, channel
- counterparty_name, counterparty_account
- transaction_time, create_time

**预警表 (t_alert)**
- alert_id, customer_id, alert_type(大额/可疑/名单命中)
- risk_score, risk_level, triggered_rules(JSON)
- status(待处理/处理中/已确认/已排除)
- assigned_to, assigned_time
- process_result, process_time

**案件表 (t_case)**
- case_id, alert_id, customer_id
- case_status(调查中/待审批/已报送/已结案)
- str_id(关联可疑交易报告)
- investigator, reviewer, approver
- create_time, close_time
