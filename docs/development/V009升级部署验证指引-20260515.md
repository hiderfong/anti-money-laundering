## 三、验证路径 A：旧库升级到 V009

该路径用于验证真实升级。请不要在第 3 步和第 4 步之间删除 MySQL 数据卷。

### 1. 拉取仓库

```bash
git clone https://github.com/hiderfong/anti-money-laundering.git
cd anti-money-laundering
```

### 2. 启动 MySQL

```bash
docker compose -f docker/docker-compose-dev.yml up -d mysql
```

等待 MySQL 就绪：

```bash
until docker exec aml-mysql mysqladmin ping -h 127.0.0.1 -uroot -pCHANGE_ME_DEV_DB_PASSWORD --silent; do
  sleep 2
done
```

### 3. 使用 V008 基线初始化旧库

```bash
git checkout fc45787944f3d20738ba73fa36912ad26c8a16c7

JAVA_HOME=$(/usr/libexec/java_home -v 21) \
mvn spring-boot:run \
  -Dspring-boot.run.profiles=dev,no-redis \
  -Dspring-boot.run.arguments="--aml.redis.enabled=false --aml.neo4j.enabled=false"
```

启动成功后停止应用，保留 MySQL 数据卷。

检查 Flyway 当前版本应不高于 V008：

```bash
docker exec aml-mysql mysql -uroot -pCHANGE_ME_DEV_DB_PASSWORD aml_system \
  -e "SELECT version, success FROM flyway_schema_history ORDER BY installed_rank;"
```

### 4. 切换到 V009 目标基线并启动

```bash
git checkout f87dd631280aebe6b145a78cb5573ee158418b3a

JAVA_HOME=$(/usr/libexec/java_home -v 21) \
mvn spring-boot:run \
  -Dspring-boot.run.profiles=dev,no-redis \
  -Dspring-boot.run.arguments="--aml.redis.enabled=false --aml.neo4j.enabled=false"
```

### 5. 验证 V009 已应用

```bash
docker exec aml-mysql mysql -uroot -pCHANGE_ME_DEV_DB_PASSWORD aml_system \
  -e "SELECT version, description, success FROM flyway_schema_history WHERE version='9';"
```

期望：

```text
version | description                      | success
9       | add rectification audit columns   | 1
```

验证字段存在：

```bash
docker exec aml-mysql mysql -uroot -pCHANGE_ME_DEV_DB_PASSWORD aml_system \
  -e "SHOW COLUMNS FROM t_rectification_task WHERE Field IN ('created_by','updated_by');"
```

期望至少返回：

```text
created_by
updated_by
```

### 6. 验证整改任务创建接口

登录：

```bash

  -H 'Content-Type: application/json' \
```

创建整改任务：

## 五、失败排查

| 现象 | 可能原因 | 排查命令 |
|------|----------|----------|
| `POST /api/rectifications` 返回 500 | V009 未执行或字段不存在 | `SHOW COLUMNS FROM t_rectification_task` |
| 登录返回 401 | 使用了旧密码 | 当前管理员密码为 `admin123` |
| Flyway 未执行 V009 | 启动的不是目标基线或连接到错误数据库 | `SELECT * FROM flyway_schema_history` |
| E2E 数据计数为 0 | `E2E_RUN_ID` 与验证数据不一致 | 使用同一 shell 执行 seed 和 verify |
| cleanup 出现转义错误 | 使用旧版清理脚本 | 确认脚本包含 `ESCAPE '!'` |
