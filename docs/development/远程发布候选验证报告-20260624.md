## 6. 本轮发现并修复的问题

### 6.1 产品列表远程 500

远程烟测发现：

- 接口：`GET /api/products/page`
- 现象：HTTP 500
- 日志：`Cannot invoke "Object.hashCode()" because "pk" is null`
- 原因：远程旧产品数据中 `payment_mode` 为 `NULL`，代码使用 `Map.of(...).getOrDefault(null, ...)` 时触发 JDK 不接受空 key 的异常。

修复：

- 文件：`src/main/java/com/insurance/aml/module/product/service/impl/ProductServiceImpl.java`
- 做法：新增 null-safe label 映射方法，`productType` 和 `paymentMode` 为空时使用默认文案。
- 测试：新增 `ProductServiceImplTest`，覆盖 `paymentMode` 为空和 `productType` 为空的旧数据场景。
- 验证：GitHub Release Gate 全绿，远程重新部署后 `/api/products/page` 通过。

### 6.2 烟测脚本响应兼容

远程验证临时脚本最初只兼容标准 `Result` 包裹响应；`STR` 和部分列表接口返回未包裹分页或数组结构，导致脚本误判。已在临时脚本中兼容包裹和未包裹响应。本项未改仓库代码。
