# 性能基线与 E2E 可视化报告说明

> 日期：2026-05-15
> 目标：让发布验证不只给出 PASS/FAIL，还能留下可浏览、可比较的证据。

## 一、E2E HTML 报告

`scripts/collect-e2e-artifacts.sh` 现在会同时生成：

| 文件 | 说明 |
|------|------|
| `e2e-summary.md` | Markdown 摘要，适合 GitHub 页面直接阅读 |
| `e2e-summary.html` | HTML 可视化报告，包含步骤状态、日志链接和截图预览 |
| `manifest.txt` | artifact 文件清单 |
| `logs/backend.log` | 后端运行日志 |
| `logs/frontend.log` | 前端运行日志 |
| `screenshots/*` | 浏览器 E2E 截图 |

GitHub Actions 中 `e2e-release-gate-artifacts` artifact 会自动包含 HTML 报告。

## 二、性能基线脚本

新增脚本：

```bash
bash scripts/performance-baseline.sh
```

默认配置：

| 环境变量 | 默认值 | 说明 |
|----------|--------|------|
| `BASE_URL` | `http://127.0.0.1:8080/api` | 后端 API 地址 |
| `PERF_ARTIFACT_DIR` | `/tmp/aml-performance-baseline` | 输出目录 |
| `ITERATIONS` | `20` | 每个端点采样次数 |
| `WARMUP` | `2` | 每个端点预热次数 |
| `USERNAME` | `admin` | 登录用户名 |
| `PASSWORD` | `admin123` | 登录密码 |
| `TOKEN` | 空 | 已有 token 时可跳过登录 |
| `BASELINE_ENDPOINTS_FILE` | 空 | 自定义端点清单 |

默认端点：

| 名称 | 路径 | 是否认证 |
|------|------|----------|
| `system-health` | `/system/health` | 否 |
| `dashboard-overview` | `/dashboard/overview` | 是 |
| `customers-page` | `/customers/page?pageNum=1&pageSize=20` | 是 |
| `transactions-page` | `/transactions/page?pageNum=1&pageSize=20` | 是 |
| `alerts-page` | `/alerts/page?pageNum=1&pageSize=20` | 是 |
| `cases-page` | `/cases/page?pageNum=1&pageSize=20` | 是 |

输出文件：

| 文件 | 说明 |
|------|------|
| `performance-baseline.csv` | 适合后续机器比对 |
| `performance-baseline.md` | 适合人工审阅 |

## 三、自定义端点

可创建一个端点文件：

```text
customer-detail|GET|/customers/1|true
rule-feedback|GET|/monitoring/rules/feedback/summary|true
```

然后执行：

```bash
BASELINE_ENDPOINTS_FILE=/path/to/endpoints.txt ITERATIONS=50 bash scripts/performance-baseline.sh
```

格式为：

```text
name|HTTP_METHOD|path|auth_required
```

## 四、建议门槛

正式性能 SLA 仍需结合真实部署资源确认。当前建议先以趋势基线为主：

| 场景 | 建议 P95 |
|------|----------|
| 健康检查 | < 100ms |
| Dashboard overview | < 500ms |
| 1k 数据列表分页 | < 800ms |
| 10k 数据列表分页 | < 1500ms |
| 图分析接口 | 单独建图分析专项基线 |
