# 反洗钱系统 — 功能拆分与开发任务列表

> 本文档是 PRD 的工程化拆解，供多 Agent 协同开发使用。  
> 每个任务有唯一编号、明确的输入/输出、验收标准和依赖关系。  
> 版本：v1.0 | 日期：2025年5月

---

## 一、模块总览与开发阶段划分

### 1.1 开发阶段

| 阶段 | 名称 | 周期 | 包含模块 | 目标 |
|------|------|------|---------|------|
| P0 | 基础设施 | 第1-2周 | 项目骨架、数据库、公共组件 | 框架可运行，所有基础组件就绪 |
| P1 | 核心能力 | 第3-6周 | KYC、名单筛查、交易监测、报告 | 核心合规功能可用 |
| P2 | 流程闭环 | 第7-9周 | 预警管理、案件管理、监管报送 | 端到端业务流程打通 |
| P3 | 增强功能 | 第10-12周 | 产品评估、自评估、仪表盘、报表 | 完整系统交付 |

### 1.2 模块依赖关系

```
P0-基础设施
  ├── P1-KYC/CDD
  ├── P1-名单筛查 ──────┐
  ├── P1-交易监测 ──────┤
  │                      ├── P2-预警管理
  │                      ├── P2-案件管理
  │                      └── P2-监管报送
  ├── P3-产品风险评估
  ├── P3-风险自评估
  └── P3-仪表盘与报表
```

## 七、Agent 协作规范

### 7.1 任务认领规则

1. 每个 Agent 认领任务前先检查依赖任务是否已完成
2. 认领时在任务状态中标注 `IN_PROGRESS + AgentID`
3. 完成后标注 `DONE + AgentID + 完成日期`

### 7.2 代码规范

- 分支命名：`feature/{任务编号}-{简述}`，如 `feature/P1-01-01-customer-entity`
- Commit 格式：`[{任务编号}] {描述}`
- 每个任务必须包含对应单元测试
- API 接口必须编写 Swagger 注解

### 7.3 交付物检查清单

每个任务完成后必须提供：

- [ ] 代码已提交到对应分支
- [ ] 单元测试通过
- [ ] API 文档（Swagger 注解）
- [ ] 数据库变更（如有）已加入 Flyway 迁移脚本
- [ ] README 或注释说明

### 7.4 任务状态标记

```
TODO         → 未开始
IN_PROGRESS  → 进行中（附 AgentID）
BLOCKED      → 被阻塞（附阻塞原因）
DONE         → 已完成（附 AgentID + 日期）
REVIEW       → 待审核
```
