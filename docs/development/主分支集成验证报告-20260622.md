E2E 首次失败未进入业务测试阶段，失败点为 `Install Playwright Chromium headless shell`，GitHub Runner 在 `npm exec playwright install --only-shell chromium` 下载浏览器时超时并终止进程。

已追加修复：

- 在 GitHub workflow 中新增系统浏览器检测。
- 若 runner 已安装 Google Chrome，则设置 `PLAYWRIGHT_CHANNEL=chrome` 并跳过 Playwright 浏览器下载。
- 若 runner 只有 Chromium，则设置 `PLAYWRIGHT_EXECUTABLE_PATH` 并跳过下载。
- 仅当 runner 没有系统浏览器时，才回退执行 `npx playwright install-deps chromium` 与 `npx playwright install chromium`。

该修复不改变业务测试脚本，只调整 CI 浏览器准备策略，目标是降低远程 E2E 对 Playwright CDN 下载稳定性的依赖。
