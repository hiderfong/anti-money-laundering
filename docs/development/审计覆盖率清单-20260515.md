## 二、优先补齐清单

P1：发布后优先补齐，均属于合规留痕关键操作。

| 模块 | 接口类别 | 建议审计动作 |
|------|----------|--------------|
| STR 可疑交易报告 | 创建、提交复核、审核、报送监管 | `CREATE/UPDATE/APPROVE/SUBMIT` |
| 大额交易报告 | 生成、审核、报送、重试失败报送 | `CREATE/APPROVE/SUBMIT` |
| 整改中心 | 创建整改、更新进度、状态变更、验收 | `CREATE/UPDATE/APPROVE` |
| 特别预防措施 | 名单更新、回溯筛查、冻结扣划、升级预警/案件 | `CREATE/UPDATE/ESCALATE` |
| 调查协查 | 创建协查、状态变更、追加动作 | `CREATE/UPDATE` |
| 系统用户 | 创建、更新、删除、重置密码、分配角色 | `CREATE/UPDATE/DELETE` |
| 规则配置 | 新增、更新、启用、停用 | `CREATE/UPDATE` |

## 三、可暂缓或按需记录

| 模块 | 说明 |
|------|------|
| 登录/刷新/登出 | 当前由认证日志和限流覆盖；如监管要求登录留痕，可补 `SECURITY` 类型审计 |
| 通知已读 | 业务风险低，可暂缓 |
| 审计日志查询 | 本身是查询行为，通常只对导出动作做审计 |
| 健康检查、字典查询、Dashboard 查询 | 只读接口，不纳入写操作审计覆盖率 |

## 四、验收标准

每补齐一个模块，应满足：

- Controller 写接口标注 `@AuditLog`。
- 审计日志包含 `traceId`、用户、IP、URI、请求方法、操作类型、耗时、结果。
- 请求参数中密码、证件号、手机号、银行卡号等敏感字段已脱敏。
- 对审核、报送、状态流转类接口，审计描述能区分动作语义。
