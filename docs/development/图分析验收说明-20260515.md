# 图分析验收说明

> 日期：2026-05-15  
> 目标：把环形交易、多层转账、共同账户、网络密度分析纳入可验证闭环。

---

## 一、当前状态

图分析接口位于 `/api/monitoring/graph/*`，由 `aml.neo4j.enabled=true` 控制启用。

| 能力 | 接口 | 返回对象 | 验收方式 |
|------|------|----------|----------|
| 环形交易检测 | `GET /monitoring/graph/ring-detection?customerId={id}` | `RingTransactionResult` | 返回 `code=200`，包含 `detected/pathNodes` |
| 多层转账追踪 | `GET /monitoring/graph/multi-layer-transfer?customerId={id}&maxDepth=3` | `MultiLayerTransferResult` | 返回 `code=200`，包含 `chains/maxDepth/suspicious` |
| 共同账户检测 | `GET /monitoring/graph/shared-accounts?customerId={id}` | `SharedAccountResult` | 返回 `code=200`，包含 `sharedAccounts` |
| 网络密度检测 | `GET /monitoring/graph/network-density?customerId={id}&densityThreshold=10` | `NetworkDensityResult` | 返回 `code=200`，包含 `relatedCustomerCount/transactionCount` |

---

## 二、E2E 验收开关

默认 CI 不启用 Neo4j，因此主 E2E 不强制执行图分析接口。真实依赖栈验证时可开启：

```bash
export AML_NEO4J_ENABLED=true
export RUN_GRAPH_E2E=true
bash scripts/e2e-test.sh
```

启用后，`scripts/e2e-test.sh` 会在交易录入后额外验证 4 个图分析接口。

---

## 三、验收前置条件

1. 后端启动参数包含 `--aml.neo4j.enabled=true`。
2. Neo4j 服务可用。
3. 测试交易已写入，并同步到 Neo4j。
4. 使用具备交易监测查看权限的账号登录。

---

## 四、通过标准

| 项目 | 标准 |
|------|------|
| 接口可达 | 4 个图分析接口均返回 `code=200` |
| 空结果容错 | 无图路径时返回空数组或 `detected=false`，不得返回 500 |
| 数据结构 | 前端 TypeScript 类型与后端 DTO 对齐 |
| 日志 | 后端日志能看到图分析请求和执行结果 |
| 可选启用 | `aml.neo4j.enabled=false` 时主 E2E 不受影响 |
