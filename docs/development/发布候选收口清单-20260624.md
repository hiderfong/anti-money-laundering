## 4. 远程部署执行步骤

远程服务器当前历史记录显示曾采用 `prod,no-es` + systemd 托管后端，且公网由 Nginx/HTTPS 代理到本机后端。建议本轮优先沿用该形态，避免在同一发布窗口内同时切换部署架构。

### 4.1 备份

```bash
cd /work/aml
mkdir -p backups/pre-rc1-$(date +%Y%m%d%H%M%S)

# 备份当前工作区与生产配置。生产配置备份必须保留在服务器本地，不提交仓库。
tar --exclude='.git' --exclude='target' --exclude='node_modules' \
  -czf backups/pre-rc1-$(date +%Y%m%d%H%M%S)/worktree.tgz .

# 按远程实际数据库配置执行数据库备份。
mysqldump -h "$AML_DB_HOST" -P "$AML_DB_PORT" -u "$AML_DB_USER" -p"$AML_DB_PASSWORD" \
  --single-transaction --routines --triggers "$AML_DB_NAME" \
  > backups/pre-rc1-$(date +%Y%m%d%H%M%S)/mysql.sql
```

### 4.2 同步代码

```bash
cd /work/aml
git fetch github main
git checkout main
git reset --hard d4aba683137f9098bfc96e0108c649e7642beeec
```

如远程仓库只有 `origin` 远端，应先确认 `origin/main` 指向 GitHub `main` 最新提交，再执行同等操作。

### 4.3 配置与预检

```bash
cd /work/aml
chmod 600 /work/aml/.env.production
bash scripts/prod-readiness-check.sh /work/aml/.env.production
bash scripts/deploy-preflight.sh /work/aml/.env.production --skip-compose
```

如果目标环境使用正式 Docker Compose 形态，则不加 `--skip-compose`，并确保 `docker/docker-compose.yml` 能正常渲染。

### 4.4 构建与重启

systemd/JAR 形态：

```bash
cd /work/aml
mvn -q -DskipTests package
AML_WORK_DIR=/work/aml \
AML_ENV_FILE=/work/aml/.env.production \
AML_JAR_PATH=/work/aml/target/aml-system-1.0.0-SNAPSHOT.jar \
AML_SERVICE_NAME=aml-system \
sudo -E bash scripts/install-systemd-service.sh
```

Docker Compose 形态：

```bash
cd /work/aml
ENV_FILE=/work/aml/.env.production bash scripts/deploy.sh v0.9.0-rc1
```

## 6. 回滚方案

### 6.1 应用回滚

优先回滚应用版本，不优先回滚数据库。

```bash
cd /work/aml
git reset --hard <previous-good-commit>
mvn -q -DskipTests package
sudo systemctl restart aml-system
curl -s http://127.0.0.1:8080/api/system/health
```

Docker Compose 形态可改为：

```bash
ENV_FILE=/work/aml/.env.production bash scripts/deploy.sh <previous-version>
```

### 6.2 数据库回滚

仅当应用回滚后仍无法恢复，且确认新增迁移造成不可接受影响时，才执行数据库恢复：

```bash
mysql -h "$AML_DB_HOST" -P "$AML_DB_PORT" -u "$AML_DB_USER" -p"$AML_DB_PASSWORD" "$AML_DB_NAME" \
  < /path/to/pre-rc1/mysql.sql
```

数据库恢复前必须二次确认当前线上写入窗口、用户影响和备份文件完整性。
