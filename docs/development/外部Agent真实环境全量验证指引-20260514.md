# 外部 Agent 真实环境全量验证指引 - 20260514

## 目标

本指引用于让沙箱外 AI Agent 在本机真实依赖环境中，按固定顺序验证当前项目改动，并生成可回传的验证报告。

重点验证范围：

- 后端能否在真实 MySQL 上启动。
- Flyway 是否成功执行到 `V008__investigation_assistance.sql`。
- `V007` 特别预防、整改中心和 `V008` 调查协查相关表、权限、接口是否可用。
- 后端 API E2E、前端 HTTP smoke、浏览器 E2E 是否全量通过。

## 执行原则

- 只执行验证，不修改源代码、迁移脚本或业务配置。
- 每条关键命令都要保存输出，失败时保留日志和错误上下文。
- 若某一步失败，不要直接修复代码；继续收集可用证据，并在报告中写明阻塞点。
- 只允许停止本次验证过程中由外部 Agent 启动的进程。

## 6. 验证 Flyway 和数据库结构

后端启动成功后执行：

```bash
mysql -uroot -paml_dev_123 aml_system -e "
SELECT installed_rank, version, script, success
FROM flyway_schema_history
ORDER BY installed_rank;
" 2>&1 | tee "$LOG_DIR/flyway-history.log"

mysql -uroot -paml_dev_123 aml_system -e "
SELECT table_name
FROM information_schema.tables
WHERE table_schema = DATABASE()
  AND table_name IN (
    't_watchlist_update_job',
    't_retrospective_screening_job',
    't_special_measure',
    't_freeze_seizure_deduction',
    't_investigation_request',
    't_investigation_action'
  )
ORDER BY table_name;
" 2>&1 | tee "$LOG_DIR/new-tables.log"

mysql -uroot -paml_dev_123 aml_system -e "
SELECT permission_code, permission_name, type, path, status
FROM t_permission
WHERE permission_code IN (
  'MENU_SPECIAL_PREVENTION',
  'special:view',
  'special:manage',
  'MENU_RECTIFICATION',
  'rectification:view',
  'rectification:manage',
  'MENU_INVESTIGATION',
  'investigation:view',
  'investigation:manage'
)
ORDER BY permission_code;
" 2>&1 | tee "$LOG_DIR/new-permissions.log"

mysql -uroot -paml_dev_123 aml_system -e "
SELECT r.role_code, p.permission_code
FROM t_role r
JOIN t_role_permission rp ON rp.role_id = r.id
JOIN t_permission p ON p.id = rp.permission_id
WHERE p.permission_code IN (
  'MENU_SPECIAL_PREVENTION',
  'special:view',
  'special:manage',
  'MENU_RECTIFICATION',
  'rectification:view',
  'rectification:manage',
  'MENU_INVESTIGATION',
  'investigation:view',
  'investigation:manage'
)
ORDER BY r.role_code, p.permission_code;
" 2>&1 | tee "$LOG_DIR/new-role-permissions.log"
```

预期：

- `flyway_schema_history` 中 `V007`、`V008` 均存在且 `success=1`。
- 新增 6 张表全部存在。
- 新增菜单和按钮权限全部存在。
- admin、compliance、investigator 等角色拥有对应权限，具体以迁移脚本授权结果为准。
