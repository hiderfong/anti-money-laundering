## 四、安全框架

### 4.1 认证授权

| 项目 | 选型 | 说明 |
|------|------|------|
| 认证框架 | Spring Security 6.x | 主体框架 |
| 认证方式 | JWT（无状态） | Access Token + Refresh Token |
| Token 库 | jjwt / java-jwt | JWT 生成和验证 |
| 密码加密 | BCrypt | Spring Security 内置 |
| 权限模型 | RBAC | 角色-权限-数据行级 |

### 4.2 数据安全

| 项目 | 方案 | 说明 |
|------|------|------|
| 敏感字段加密 | AES-256-GCM | 证件号、手机号等，应用层加密存储 |
| 数据脱敏 | 自定义注解 @MaskField | 姓名、证件号、手机号展示脱敏 |
| 传输加密 | HTTPS / TLS 1.2+ | 生产环境强制 |
| SQL 注入防护 | MyBatis 参数绑定 | 禁止拼接 SQL |
| 操作审计 | AOP + @AuditLog | 全操作可追溯 |

### 6.2 代码分层规范

| 层 | 命名规范 | 职责 |
|------|---------|------|
| Controller | XxxController | 接口定义、参数校验、调用 Service |
| Service | XxxService / XxxServiceImpl | 业务逻辑、事务管理 |
| Mapper | XxxMapper (extends BaseMapper) | 数据访问 |
| Entity | Xxx (t_xxx 表映射) | 数据库实体 |
| DTO | XxxRequest / XxxResponse / XxxVO | 数据传输对象 |
| VO | XxxVO | 视图对象（返回前端） |
| Converter | XxxConverter (MapStruct) | 对象转换 |

### 6.3 API 规范

| 规范 | 说明 |
|------|------|
| 风格 | RESTful |
| 响应格式 | `{"code": 200, "message": "success", "data": {...}, "timestamp": ...}` |
| 分页参数 | `page`, `size`, `sort` |
| 分页响应 | `{"total": 100, "list": [...], "page": 1, "size": 20}` |
| 错误码 | 业务错误码 6 位数字，按模块分段 |
| 文档 | Knife4j（Swagger 增强），所有接口必须写注解 |

### 6.4 错误码分段

| 段 | 模块 | 示例 |
|------|------|------|
| 100000-109999 | 系统/认证 | 100001 用户名或密码错误 |
| 110000-119999 | KYC/客户 | 110001 客户不存在 |
| 120000-129999 | 名单筛查 | 120001 名单导入格式错误 |
| 130000-139999 | 交易监测 | 130001 规则配置无效 |
| 140000-149999 | 预警 | 140001 预警状态不允许该操作 |
| 150000-159999 | 案件 | 150001 案件已结案 |
| 160000-169999 | 报送 | 160001 XML 格式校验失败 |

    image: redis:7-alpine
    ports: ["6379:6379"]

    image: bitnami/kafka:3.6

    image: elasticsearch:8.12.0

## 八、技术选型决策依据

| 决策 | 选型 | 替代方案 | 选择理由 |
|------|------|---------|---------|
| 语言 | Java 21 | Python / Go | 企业级生态成熟、Virtual Threads 高并发、团队技能匹配 |
| 框架 | Spring Boot 3.4 | Quarkus / Micronaut | 生态最完善、社区最大、学习资料多 |
| ORM | MyBatis-Plus | JPA/Hibernate | SQL 可控、适合复杂查询、国内团队习惯 |
| 规则引擎 | Drools + Redis Lua | Easy Rules / 自研 | Drools 处理复杂规则，Redis 处理速度规则，互补 |
| 数据库 | MySQL 8.0 | PostgreSQL / Oracle | 公司基础设施统一、运维团队熟悉 |
| 搜索 | Elasticsearch | Solr / Meilisearch | 名单模糊匹配性能优异、生态成熟 |
| 消息队列 | Kafka | RabbitMQ / RocketMQ | 高吞吐、交易事件流场景适合 |
| 缓存 | Redis | Memcached | 数据结构丰富、支持 Lua 脚本 |
| JWT | jjwt | java-jwt | Spring 生态兼容好 |
