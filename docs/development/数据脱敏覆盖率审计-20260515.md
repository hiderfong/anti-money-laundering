# 数据脱敏覆盖率审计

> 日期：2026-05-15
> 目标：确认敏感字段在 API 返回、审计日志和 E2E 证据中默认不明文暴露。

## 一、当前能力

项目已有两套脱敏机制：

| 机制 | 位置 | 适用场景 |
|------|------|----------|
| `@MaskField` + `MaskAspect` | Controller 返回值 AOP | API 响应对象、分页对象、集合对象 |
| `@Sensitive` + `SensitiveSerializer` | Jackson 序列化器 | 任意 JSON 序列化场景 |

本轮增强：

- `MaskAspect` 已支持递归处理 `Result -> PageResult -> list`、集合、数组和 Map。
- `CustomerVO.email` 增加邮箱脱敏。
- `UserVO.email`、`UserVO.phone` 增加脱敏。
- `TransactionVO.counterpartyAccount` 增加银行卡/账号脱敏。

## 二、已覆盖字段

| 对象 | 字段 | 脱敏类型 |
|------|------|----------|
| `CustomerVO` | `name` | 姓名 |
| `CustomerVO` | `idNumber` | 证件号 |
| `CustomerVO` | `phone` | 手机号 |
| `CustomerVO` | `email` | 邮箱 |
| `CustomerBeneficialOwnerVO` | `ownerName` | 姓名 |
| `CustomerBeneficialOwnerVO` | `ownerIdNumber` | 证件号 |
| `UserVO` | `email` | 邮箱 |
| `UserVO` | `phone` | 手机号 |
| `TransactionVO` | `counterpartyAccount` | 银行卡/账号 |

## 三、仍需审计的字段

| 风险点 | 当前状态 | 建议 |
|--------|----------|------|
| `AlertVO.customerName` | 未脱敏 | 预警列表是否展示明文客户名需由产品确认；默认建议列表脱敏、详情按权限展示 |
| `CaseVO.customerName` | 未脱敏 | 案件列表建议脱敏；调查详情可按权限展示 |
| `LargeTxnReportVO.customerName` | 未脱敏 | 报告列表建议脱敏；XML 原文仅限报送权限 |
| `LargeTxnReportVO.counterpartyInfo` | JSON 字符串 | 建议后续改为结构化 VO 后分别脱敏账号/名称 |
| 导出文件 | 需逐项核查 | Excel/CSV 导出应和页面同等脱敏，或仅允许高权限导出明文 |
| 审计日志请求参数 | 已有基础脱敏 | 建议补充针对 JSON 字符串中的证件号、手机号、账号的正则脱敏 |

## 四、验收建议

建议后续补一组接口级测试：

1. 创建含手机号、邮箱、证件号的客户。
2. 查询客户详情和客户分页，断言返回值已脱敏。
3. 查询用户分页，断言邮箱和手机号已脱敏。
4. 查询交易分页，断言交易对手账号已脱敏。
5. 导出文件中不出现完整证件号、手机号、银行卡号。

## 五、产品规则建议

| 页面 | 默认显示 | 明文查看 |
|------|----------|----------|
| 客户列表 | 脱敏 | 高权限点击查看 |
| 客户详情 | 脱敏 | 高权限点击查看并记录审计 |
| 预警列表 | 脱敏 | 处理人/调查员可查看 |
| 案件详情 | 按案件权限展示 | 查看明文应记录审计 |
| 报送 XML | 不默认展示 | 报送权限 + 审计记录 |
